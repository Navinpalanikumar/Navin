50K Orders / Export 14–16s → Sub-second AWS + Azure

What Made This Perfect

  • Composite DB index on (company_id, order_id) on 74M-row table — batch query from 14–16s to sub-second
  • S3 pointer pattern avoids Kafka 1MB limit: 5000 UUIDs stored in S3, lightweight index key in message

  • Cloud-agnostic: @ConditionalOnProperty(matchIfMissing=false) prevents cross-cloud bean loading
